Telemetry Data Archives Settings Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ is able to archive the telemetry data on the basis of files. More than one file can be specified for archiving and together with a variety of options the data can be stored in many different ways. In general, the telemetry data is archived in a raw (unprocessed) form that allows the replay at a later time even with a modified database. However, in order to be compatible with incremental telemetry data streams (for example the CCSDS Packet Telemetry Standard) and to guarantee correct retrievals, the values of all telemetry parameters are embedded into the archive file at a definable interval.
Retrievals are performed through the various displays and can take place at the same time as the real-time processing. Due to the fact that the telemetry data can be distributed over many files, it is possible to determine a default retrieval file and to change to another one for each retrieval request separately.
Furthermore, the implemented backup capabilities simplify the way how to handle long-term archiving. Sometimes it may also be advantageous to extract values for a set of parameters over a specified amount of time. This is a supported feature and particularly useful to process the extracted telemetry data offline, for example, inside a spreadsheet. Extractions are not limited to specified parameter sets but can also include complete portions of history files originating from multiple sources. With this capability, interesting periods of telemetry data can be extracted and distributed as a fully functional history file.

How do I...

Configure the Telemetry Data Archives

  1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
  2. Select the Archivals page.

  3. Read more about:

    Adding a new File to the Archives

    1. Type the file name into the edit field of the Files combo box. Use the Browse... button to search for an existing archive file to be added.
    2. Push the Add button in order to add it to the list of archive files.

    3. Configure the archive options and behavior for the file just added.

      1. Mark the Perform Archiving check box inside the Behavior group box if archiving should take place on that file.
      2. Choose between the available archiving schedule options by checking the corresponding radio button:

        Option

        Description

        Always

        Perform continuous archiving on the file.

        Daily between t1 and t2

        Restrict the archiving to a daily period between t1 and t2.

        From t1 to t2

        Limits the archiving to a time between t1 and t2.
      3. Select the Use filename in case of failures check box if a replacement file with name filename should be created and used to archive telemetry data that could not be saved on the original one.
        Note:
        The telemetry data archiving is continued on the original file as soon as the problem is resolved.
      4. Mark the Audit Failures check box if a failure (during archiving) should be reported inside the Global Eventbox.
      5. Mark the Show Status check box if the archiving status of the file should be shown on the status bar.
        Note:
        If multiple archive files are in use with this option enabled, the status of that one is shown that has the least amount of storage capacity available. The indicated amount of storage capacity (in percent) reflects the available space on the media.
      6. Choose one of the available archiving policies by checking the corresponding radio button inside the Properties group box:

        Option

        Description

        Cyclic

        Archiving is done on a cyclic basis i.e. whenever the specified file size is reached or no space on the storage media is available any more the file is incrementally overwritten from its beginning.

        Noncyclic

        Archiving is performed until the specified file size is reached or no space is available any more on the storage media. No further actions are performed on that file afterwards.

        From t1 to t2

        Limits the archiving to a time between t1 and t2.
      7. Specify the size of the file by choosing on of these options:

        Option

        Description

        Unlimited

        All the available space on the storage media is used up for archiving.

        Restricted to n MB

        Archiving is performed until the specified file size of n MB is reached.
      8. Specify the interval measured in telemetry packets after which the values of all parameters should be added to the file by selecting the Updates all n Units check box and entering the interval n into the edit field.
        Note:
        This option is only required when incremental telemetry data streams like those compatible with the Packet Telemetry Standard (CCSDS) for example are archived.
      9. Mark the Use Compression check box if a file based compression should take place during archiving.
        Note:
        This option is only effective on storage medias supporting file compression.
      10. Push the Options... button in order to specify telemetry data related retention options for the archives.
        The Telemetry Data Archival Options Dialog Box will appear.
        Read more about:

        Specifying Archival Retention Options

        These archival retention options are available to find an optimal trade-off between space requirements and retrieval efficiency and they apply to all files in the archives. They control the contents of the full updates which are always performed after n telemetry source packets or formats. The archiving of the raw telemetry source packet stream however is not affected.
        It should also be pointed out that by specifying a limited number of n days to archive a telemetry source packet or telemetry parameter that data will be lost for any retrievals starting n days after their last update.

        1. Specify the archiving behavior for all or a specific telemetry source packet (or format) by selecting either the item All telemetry source packets or the name of an individual one in the upper combo box.
        2. Push either the Forever radio button to keep the chosen telemetry source packets (or formats) in the cache all the time or for a limited amount of days only by checking the n days only radio button and entering the number n of days.
          Note:
          Specifying a retention of 0 days does not archive the telemetry source packet at all (in the full cache updates).
        3. Repeat the previous two steps in order to specify the retention options for more than one specific telemetry source packet (or format).
          Note:
          The current settings can always be reset by re-selecting the item 'All telemetry source packets' and by specifying the appropriate option.
        4. Specify the archiving behavior for all or a specific telemetry parameter by selecting either the item All telemetry parameters or the name of an individual one in the lower combo box.
        5. Push either the Forever radio button to keep the chosen telemetry parameters in the cache all the time or for a limited amount of days only by checking the n days only radio button and entering the number n of days.
          Note:
          Specifying a retention of 0 days does not archive the telemetry parameter at all (in the full cache updates).
        6. Repeat the previous two steps in order to specify the retention options for more than one specific telemetry parameter.
          Note:
          The current settings can always be reset by re-selecting the item 'All telemetry parameters' and by specifying the appropriate option.
        7. Push the OK button.

    Enabling/Disabling the Archiving on a File

    1. Select the name of the archive file to be enabled or disabled in the Files combo box.
    2. Mark the Perform Archiving check box inside the Behavior group box if archiving should take place on that file or uncheck it to disable archiving.

    Resetting the Contents of a File

    1. Select the name of the archive file those contents should be reset in the Files combo box.
    2. Mark the Delete File check box.
      Note:
      The file is re-created automatically the next time any archiving takes place on it.

    Showing the Archives Status

    The current status of the telemetry data archives is displayed in a pane on the status bar.
    1. Select the name of the archive file to be included into the list of files those status should be displayed in the Files combo box.
    2. Mark the Show Status check box.
      Note:
      If multiple archive files are in use with this option enabled, the status of that one is shown that has the least amount of storage capacity available. The indicated amount of storage capacity (in percent) reflects the available space on the media.
  4. Push the OK button.

Specify the Telemetry Data Retrieval Sources

  1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
  2. Select the Retrievals page.

  3. Read more about:

    Adding a new Retrieval File

    1. Type the file name into the edit field of the Files combo box. Use the Browse... button to search for an existing retrieval file to be added. The Inspect button can be pushed to verify if a certain file name represents a valid retrieval file. All information about the inspected file is displayed inside the Source Information group box.
    2. Push the Add button in order to add it to the list of retrieval files.
      Note:
      By marking the Default File check box the currently selected file will be considered as the default one i.e. any retrieval activities will take place on that file if not specified differently.

    Removing a Retrieval File

    1. Select the name of the retrieval file to be removed in the Files combo box.
    2. Push the Remove button in order to take it away from the list of retrieval files. Use the Remove All button if no retrieval files should be available at all.

    Specifying default Retrieval Directories

    1. Check one or more of the available options inside the Automatic Browse group box:

      Option

      Description

      Any File

      Automatically adds every valid retrieval file inside the specified directory to the list of available retrieval sources.

      Timetagged Files

      Includes only time-tagged retrieval files into the list of available retrieval sources.
      Note:
      Read more about telemetry data backups to know what time-tagged files are.

      Enumerated Files

      Includes only enumerated retrieval files into the list of available retrieval sources.
      Note:
      Read more about telemetry data backups to know what enumerated files are.

      Note:
      More than one automatic retrieval directory can be specified if necessary. They have to be separated by a comma.

    Determining a default Retrieval File

    1. Select the name of the file to become the default retrieval file in the Files combo box.
      Note:
      Only one file can be assigned to be the default retrieval file. It is the one that is used for all retrievals in case no other file is specified.
    2. Mark the Default File check box.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry data retrieval sources' must be enabled to specify, change or delete the telemetry data retrieval sources.

Perform automatic Telemetry Data Backups

  1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
  2. Select the Backups page.
  3. Select the archive file whose backup characteristics and behavior should be specified or modified in the Files list box.
  4. Choose the file type by checking the appropriate radio button:

    Type

    Description

    Simple

    The backup file has the same name as the archive file: 'Backup of filename.HIS'.
    Note:
    If more than one backup file is generated with this type the name is: 'Backup (n) of filename.HIS'.

    Timetagged

    A part of the backup file name is the time key of the earliest telemetry data contained in the file: 'YYYY.MM.DD_hh.mm.ss.HIS'.

    Enumerated

    Enumerates the backup files by using the same name as the archive file: 'Filename_n.HIS'.
  5. Specify the directory where the backup files should be moved to by typing its path into the Directory edit field.
  6. Select one or more of the triggers determining the time when a backup should take place by marking the corresponding check boxes:

    Trigger

    Description

    When file reached its specified size

    Whenever the archive file reaches a predefined maximum size a backup is performed.
    Note:
    The original archive file is always reset after a backup has taken place.

    When file size is at least n MB

    Performs a backup when the archive file reached a specified size of n MB.
    Note:
    This trigger is not available for cyclic archive files.
  7. Mark the Perform Backups check box inside the Behavior group box if backups of the archive file should be enabled.
  8. Choose one or more of the behavior options by selecting the corresponding check boxes:

    Option

    Description

    No backups between t1 and t2

    Any attempt to perform a backup of this archive file is delayed to t < t1 or t > t2.

    No backups for the next n minutes

    All backups of this archive file are delayed until the amount of n minutes has expired after enabling this option.

    No backups on write protected storage medias

    Delays a backup of this archive file until a storage media is inserted which is not write protected.

    No backups on non-empty storage medias

    Prevents a backup of this archive file as long as a non-empty storage media is inserted. The backup is performed as soon as an empty storage media is inserted.
  9. Mark the On Failures retry n times check box if n attempts should be tried before discarding the file and eventually issue a warning message. When enabling this option, specify how many attempts and at what interval they should take place by filling in the appropriate edit fields.
  10. Select the Audit button in order to enable the reporting of failures by displaying a message inside the Global Eventbox.
  11. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry data archive backups' must be enabled to modify the telemetry data backup bahavior.

Generate automatic Telemetry Data Extractions and Reports

  1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
  2. Select the Extractions page.

  3. Read more about:

    Specifying the Data Source and File Format

    1. Type the name of the telemetry data extraction profile those source and format is going to be specified into the Profile edit field or select an existing one in the attached combo box.
      Note:
      Defining a profile describing all characteristics of a telemetry data extraction request is a convenient way to handle a variety of different extractions with a minimum effort.
    2. Select the history file of which the telemetry data should be extracted from in the Source combo box. Use the Browse... button in order to search for another history file than the ones listed.
    3. Choose the directory to where the extraction file should be written to by selecting the Destination Directory item in the Source/Destination combo box and by typing its name into the corresponding edit field. Use the Browse... button in order to search for an existing directory.
      Note:
      When no destination directory is specified the current directory is used as location for the extraction files.
      The name of these files is always composed as follows: Destination Directory\Profile Name_YYYYMMDD_HHMMSS.TXT where YYYY identifies the year, MM the month, DD the day of the month and HHMMSS the starting time of the extracted period.
    4. Determine the column, line and text delimiters to be used within the extraction file by selecting them in the Column, Row and Text combo boxes.
    5. Select the format of the time key to be used for the time to parameter value association in the Time combo box.
    6. Choose the character set of the resulting output file by checking either the ASCII or Unicode radio button.
    7. Mark the Title check box if an initial title row should be inserted at the beginning of the file.

    Specifying the Parameter Set

    1. Select the Contents page of the property sheet at the bottom.
    2. Select the parameter to be included into the extraction profile in the Parameter combo box.
    3. Press the Add button to add the selected telemetry parameter or push the Remove button to delete an existing parameter. Use the Remove All button to delete all parameters at once.
    4. Specify how the value of the added parameter should be formatted by pushing the corresponding radio button:

      Option

      Description

      Default

      The value is converted into a calibrated form. If there is no calibration curve attached to the parameter it is displayed as a coded value.

      Coded

      The value is formatted according to its coding and without calibrating it.

      Raw

      The value is displayed as raw bit pattern independent of its coding. It can be converted to one of the following numbering systems: Decimal, binary, octal and hexadecimal. On demand, the bit pattern can also be expressed as the one's complement.

      Status

      The current status code of a parameter is listed instead of its value. The code can be interpreted as added value composed of:

      Data Quality

      Out-of-limit Status

      Validity Code

      Good= 1
      Bad= 2

      In-limit= 4
      Soft limit= 8
      Hard limit= 16
      Delta limit= 32

      Valid= 64
      Invalid= 128
    5. Mark the Changes only check box if only changing values should be extracted for the selected parameter.
    6. Repeat the above steps (except for the first one) for each telemetry parameter to be part of the extraction profile.

    Specifying the Time Range

    1. Select the Range page of the property sheet at the bottom.
    2. Specify the time range the extraction should cover by choosing one of the following options:

      Option

      Description

      From t1 to t2

      All telemetry data available between the specified start time t1 and stop time t2 is extracted.
      Note:
      It is recommended to always declare a slightly larger period than required.

      Daily from t1 to t2

      The telemetry data is extracted on a daily basis always starting at the specified time t1 and stopping at t2.
      Note:
      It is recommended to always declare a slightly larger period than required.

      Periodically: All available time

      Extracts telemetry data at a specified periodical interval and in an intermittent way.

      Periodically: The last n minutes of the interval

      Extracts telemetry data at a specified periodical interval but only the last n minutes of every occurring period.

      Triggered: The last n minutes before the event

      Extracts the last n minutes before and/or after the occurrence of a specified telemetry event.

      Use an interval of n minutes

      Directs the extraction process to step ahead in intervals of n minutes instead of progressing on a continuous basis.
      Note:
      Depending on the kind of telemetry data standard supported the resulting steps may be smaller or larger than the specified interval.

    Specifying the Execution Time

    1. Select the Execution page of the property sheet at the bottom.
    2. Choose one of the available execution modes by checking the appropriate radio button:

      Mode

      Description

      Immediately

      The extraction process is started immediately.

      Once at t

      The extraction profile is executed once at time t.

      Daily at t

      The extraction profile is executed daily at time t.

      Starting at t1 with an interval of n minutes and stopping at t2

      The execution of the extraction profile takes place periodically with an interval of n minutes, starting at time t1 and stopping at t2.

      At event x

      Executes the extraction profile whenever the telemetry event x occurs.

      Note:
      The number of available execution modes depends on the type of time range chosen.

    Distributing the Results via FTP

    1. Select the FTP Transfers page of the property sheet at the bottom.
    2. Specify the location to where the resulting files should be transferred to by entering the address into the FTP Location edit field. The destination address must be a valid FTP site address in the format 'ftp://domain/directory' where 'domain' identifies a domain name or IP address and 'directory' an existing directory. Multiple destination addresses may be specified, if required, each of them separated by a semicolon or comma.
      Note:
      The check box Enable FTP Transfers must marked in order to enable the distribution of telemetry reports via FTP.
    3. Enter the user name and password into the corresponding Username and Password edit fields.
      The following convention related to user authentication is used:

      Username

      Password

      Username sent to FTP site

      Password sent to FTP site

      Empty

      Empty

      anonymous

      E-Mail address

      Username

      Empty

      Username

      Empty

      Username

      Password

      Username

      Password
      Note:
      If multiple destination addresses are specified, the corresponding user names and passwords should be separated by a semicolon or comma. Entering fewer user names and passwords than available destination addresses associates an empty user name and password to those addresses.
    4. Specify a TCP/IP port number by marking the Use Port check box and entering a port number if a specific port is required for the FTP transfer.
    5. Determine the behavior in case of failures by choosing one of the following options:

      Option

      Description

      Retry Never

      No action is taken when a FTP transfer was not successful.

      Retry Continuously

      Continuously retry to send the report at an interval (in minutes) specified in the Interval spin box.

      Retry n times

      As above but restricted to n retries.
    6. Choose the desired reporting behavior by checking the appropriate radio button:

      Option

      Description

      Report Nothing

      No problems at all are reported.

      Report Everything

      A message is generated whenever a problem or successful FTP transfer occurs.

      Report Failures

      Only failures during a FTP transfer are reported.

    Post-processing with Automation

    Read more about automation in order to understand how post-processing can be implemented.
    1. Select the Automation page of the property sheet at the bottom.
    2. Select the automation script to be executed each time the extraction file is generated in the Name combo box. The code of the automation script is always displayed in the Code area.
      Note:
      Choose the No script attached item if no automation script should be executed i.e. no post-processing should take place.
    3. Mark the Send via FTP (attached) check box if the generated output file should be sent to a client together with the resulting telemetry data extraction file.
      Note:
      The file generated by an automation script cannot be sent via FTP without the telemetry data extraction file.
    4. Mark the Disabled check box if the selected automation script should be attached but remain inactive. In this case the script is not executed i.e. no post-processing is performed.
  4. Push the Save button in order to save the new or modified telemetry data extraction profile.
    Use the Delete button in order to remove an existing telemetry data extraction profile or the Delete All button to remove all of them at once.
    Note:
    The Default extraction profile can never be deleted. It is reset whenever the Delete All button is pushed.
  5. Click the Submit button in order to activate the current telemetry data extraction profile i.e. to move it to the execution queue.
    Note:
    Telemetry data extraction profiles triggered by telemetry events do not need to be submitted.
  6. Push the OK button.

    Aborting executing Extraction Profiles

    1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
      Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
    2. Select the Extractions page.
    3. Push the Abort... button. This opens the Telemetry Data Extractions Abort Dialog Box.
    4. Inside the List list box all telemetry data extraction jobs are enumerated that are currently executing or waiting to be executed. Select only those jobs that should be aborted.
      Note:
      The lower Triggered List list view shows all telemetry data extraction profiles currently configured to be triggered by a telemetry event. The list is available for informational purposes only.
    5. Push the OK button.

    Note:
    In some cases the abortion of telemetry data extraction jobs has no immediate effect. This happens in situations where a re-schedule of the execution queue happened while this dialog box was open.

    Importing Extraction Profiles

    1. Open the Telemetry Data Extraction Profiles Import Dialog Box if it is not open yet:
      Push the Import... button inside the Archives panel of the Telemetry ribbon bar category.
    2. Enter the name of the file that contains all telemetry data extraction profiles to be imported into the Filename edit field. Use the Browse... button in order to search for an appropriate source file if necessary.
    3. Push the Inspect button in order to list all profiles that are contained in the specified file.
      Note:
      The Inspect button is available only when a file name is specified. With an empty Filename edit field the Browse... button is always visible.
    4. In the Directory combo box select a telemetry data extraction profile that should be imported.
    5. Push the Add button. Repeat the previous two steps if more than one profile should be imported or use the Add All button in order to import all available telemetry data extraction profiles.
      Use the Remove or Remove All button in order to remove one or all telemetry data extraction profiles from the importation list.
      Note:
      Telemetry data extraction profiles from more than one source file can be imported in one step by entering a different file name into the Filename edit field and following the steps above.
    6. Push the OK button.

    Exporting Extraction Profiles

    1. Open the Telemetry Data Extraction Profiles Export Dialog Box if it is not open yet:
      Push the Export... button inside the Archives panel of the Telemetry ribbon bar category.
    2. In the Directory combo box select a telemetry data extraction profile that should be exported.
    3. Push the Add button. Repeat the previous steps if more than one profile should be exported or use the Add All button in order to export all available telemetry data extraction profiles.
      Use the Remove or Remove All button in order to remove one or several telemetry data extraction profiles from the exportation list.
    4. Enter the name of the file that should contain all exported telemetry data extraction profiles into the Filename edit field. Use the Browse... button in order to search for an appropriate destination file if necessary.
    5. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry data archive extraction facility' must be enabled to generate automatic telemetry data extractions or reports.

Construct Telemetry Data History Files

  1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
  2. Select the Tools page.
  3. Type the file name of the history file to be constructed into the Filename edit field. Use the Browse... button in order to find an appropriate name.
  4. Add one or more source history files to be used for the construction of the specified history file by typing their full path name into the Sources edit field and by pushing the Add button for each file to be added. Use the Browse... button in order to find a source history file.
    In order to remove a specified source history file select it first in the Sources combo box and push the Remove button.
    Note:
    None of the specified source history files must have an identical path name as the history file to be constructed. Furthermore, all specified source history files must be synchronized in time otherwise a correct reconstruction is not guaranteed.
  5. Choose the format of the resulting history file(s) by checking either the Standard or Raw radio button.
    Note:
    Use the standard format in order to create a normal history file for SatView™. The raw format option allows the generation of one or multiple files in a binary or human readable form including various formatting options.
    Those options can be modified by pushing the Options... button:

    Changing the history file format options

    1. Choose either the ASCII or Binary radio button depending on the requested encoding for the resulting file(s).
    2. Inside the Format group box choose the requested radix and formatting options by checking the relevant radio buttons.
      In addition, select the Unicode check box if the output file(s) should use the unicode character set.
      Note:
      The format options are available only when ASCII encoding is requested.
    3. Extracted telemetry units can be separated by a carriage-return, an end-of-line character or a combination of both by choosing the appropriate None, CR, EOL or CR+EOL radio buttons.
      Note:
      The default for binary encoding is to insert no separator, for ASCII it is a carriage-return followed by an end-of-line sign.
    4. Sometimes it might be useful to create multiple output files i.e. one for each type of telemetry unit. This can be achieved by marking the Use a separate file for each telemetry unit type check box.
      If this option is chosen, it is also possible to specify a file prefix via the Use the file prefix check box and an extension by selecting Use the file extension check box and entering the relevant information.
    5. Push the OK button.
  6. Specify the number of seconds of interrupted telemetry data after which another source history file, if specified and available, should be inspected for the availability of telemetry data at that time by entering the appropriate number of seconds n into the Gaps are interruptions of more than n seconds spin box.
    Note:
    • The source history files are tested in the sequence listed in the Sources combo box.
    • This option is available only when the standard format is selected.
  7. Check the Updates all n units check box if a full update should occur after a specified amount of telemetry units and enter that number n into the appropriate spin box.
    Note:
    • This option is available only when the standard format is selected.
  8. Specify at what time the constructed history file should start at:

    Option

    Description

    From t

    The constructed history file will start with telemetry data at the specified time t (when possible).
    Note:
    The source history file which contains telemetry data closest to the specified time t will be selected for the extraction.

    From the beginning of the first source file

    The resulting history file will start at the same time as the first source history file specified.
  9. Specify the stop condition of the constructed history file:

    Option

    Description

    At t

    The constructed history file will stop at the specified time t (when possible).

    At n MBytes

    The resulting history file will stop when its size is bigger than n MBytes.

    When disk is full

    The construction of the history file will be stopped when the destination media is full or an error occurs.
  10. As an additional restriction it is possible to filter specified telemetry units. All those items that should be not part of the resulting file(s) can be removed from the Extract combo box by choosing them and then pushing the > button.
    Use the All button to reset the filter.
  11. If requested, specify a time interval during which this history file construction process should not execute:

    Option

    Description

    Prevent execution between t1 and t2

    The history file will not be constructed between t1 and t2.
    Note:
    A running process will be paused during the specified interval whereas a start will be delayed until after the period.
  12. Push the OK button.

  • Related Topics:

    Aborting the Construction of History Files

    1. Open the Telemetry Data Archives Settings Dialog Box if it is not open yet:
      Push the Settings button inside the Archives panel of the Telemetry ribbon bar category.
    2. Select the Tools page.
    3. Push the Abort button.
      Note:
      The Abort button is enabled only if a history file construction process is ongoing.
    4. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry data history file extraction facility' must be enabled to create historical telemetry data file containing selected periods of data.
