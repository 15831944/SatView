Telecommand Processing

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ provides various services and interfaces related to the processing of telecommands. The sub-system consists of an internal central command handler that accepts telecommands from various (distributed) sources and supports all common tasks related to the reliable release of telecommands (e.g. pre-transmission validation (PTV), pre-execution verification (PEV) and execution verification (CEV), command dispatch & release mechanism etc.).
Several command sources are supported: TC Manual Stack, TC Automatic Stack and the TC Procedures Monitor. In addition to those displays, there are other ones providing services related to the command history, the on-board buffers & memory management and diagnostic operating modes.

Telecommand Service

SatView™ comes with a sophisticated telecommand sub-system. It is built around a central command handler accepting telecommands from any number of (distributed) authorized command sources. The interface to the uplink front-end equipment is implemented via virtual devices i.e. individual software drivers that are built for the various equipment available.

Displays

SatView™ supports various types of telecommand related displays. Some of them allow the release of telecommands (including the ability to replay the telecommand history), others help to diagnose anomalies or reflect the on-board memory or status of the satellite.

Parameters

SatView™ provides a convenient interface to view the properties of telecommand parameters. Type, nature, coding of a parameter as well as other relevant information is displayed. It is also possible to initialize them with a value which is particularly useful for preparing telecommands.
