Telecommand Interlocks Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

How do I...

Creating, modifying or deleting Interlocks

  1. Open the Telecommand Interlocks Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category, select the Interlocks page in the dialog box and push the Edit... button.
  2. Enter a unique name into the Name edit field to create a new interlock or select an existing one inside the associated combo box for modification or deletion.
  3. Specify or change the scope of the interlock:

    Scope

    Description

    Global (all command sources)

    Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

    Local (same command source only)

    Blocks the local command source as long as the interlock is in a pending or failed state.
    Note:
    Enable the No wait state option if the interlock should not block when in a pending state.

    Local on specified subsystem

    Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
    Note:
    Enable the No wait state option if the interlock should not block when in a pending state.

    Local on specified telecommands

    Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
  4. Select or change the command source to host the interlock by selecting the computer in the Computer combo box and by pushing either TC Manual Stack, TC Automatic Stack or TC Procedures Monitor radio button.
  5. Choose the telecommand verification stage to affect the resulting state of the interlock by marking either the Reception (at ground station), Transmission, Acceptance, Start, Progress or Completion check box.

  6. Repeat the previous four steps if more than one interlock should be created or modified.
    Use the Remove button in order to delete an existing interlock. The Remove All button clears all of them at once.
  7. Push the OK button.

Note:
Only those interlocks that are in an idle state can be modified or deleted. If an interlock is in a success or failure state it has to be reset first. Pending interlocks can neither be modified nor deleted.

Prerequisites:
The privilege 'Browsing the telecommand options' must be enabled to view the existing interlocks and the 'Changing the telecommand options' privilege is required to create, modify, delete or reset them.
