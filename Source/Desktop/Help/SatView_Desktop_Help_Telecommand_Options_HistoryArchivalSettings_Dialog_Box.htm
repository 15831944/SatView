Telecommand Archive Options Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Telecommands released by the central command handler or those failed to be transmitted can be archived for later replay. Many archive options are available, including automatic backups, that simplify the command history management.

How do I...

Configure the Command History Archives

  1. Open the Telecommand Archive Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category, select the Configuration page in the dialog box and push the Options... button.
  2. Choose the file to which the options should be applied to in the Filename combo box.
  3. For each command archive file there is a possibility to attach another one that is updated only when the first fails. This can be achieved by marking the Use filename on failures check box and entering a valid file name into the associated edit field.
    Note:
    A command history file can temporarily be disabled (i.e. no archiving takes place) by marking the Disabled check box.
  4. Choose one of the available archiving policies by checking the corresponding radio button inside the Properties group box:

    Option

    Description

    Cyclic

    Archiving is done on a cyclic basis. Whenever the specified file size is reached or no space on the storage media is available any more the file is incrementally overwritten from its beginning.

    Noncyclic

    Archiving is performed until the specified file size is reached or no space is available any more on the storage media.
  5. Specify the size of the file by choosing on of these options:

    Option

    Description

    Unlimited

    All the available space on the storage media is used up for archiving.

    Restricted to n MB

    Archiving is performed until the specified file size of n MB is reached.
  6. Mark the Use Compression check box if a file based compression should take place during archiving.
    Note:
    This option is only effective on storage medias supporting file compression.
  7. Choose one of the available backup options by checking the corresponding radio button inside the Backup group box:

    Type

    Description

    Simple

    The backup file has the same name as the archive file: 'Backup of filename.TCH'.
    Note:
    If more than one backup file is generated with this type the name is: 'Backup (n) of filename.TCH'.

    Timetagged

    A part of the backup file name is the time key of the earliest telemetry data contained in the file: 'YYYY.MM.DD_hh.mm.ss.TCH'.

    Enumerated

    Enumerates the backup files by using the same name as the archive file: 'Filename_n.TCH'.
  8. Specify the directory where the backup files should be moved to by typing its path into the Directory edit field.
  9. Select one or more of the triggers determining the time when a backup should take place by marking the corresponding check boxes:

    Trigger

    Description

    When file reached its specified size

    Whenever the archive file reaches its predefined size a backup is performed.
    Note:
    The archive file is always deleted after a backup has taken place.

    When file size is at least n MB

    Performs a backup when the archive file reached a specified size of n MB.
    Note:
    This trigger is not available for cyclic archive files.
  10. Mark the Perform Backups check box if backups of the archive file should be done.
  11. Choose one or more of the behavior options by selecting the corresponding check boxes:

    Option

    Description

    No backups between t1 and t2

    Any attempt to perform a backup of this archive file is delayed to t < t1 or t > t2.

    No backups on non-empty storage medias

    Prevents a backup of this archive file as long as a non-empty storage media is inserted. The backup is performed later when an empty storage media is inserted.
  12. Mark the On Failures retry n times check box if n attempts should be tried before discarding the file and eventually issue a warning message. When enabling this option specify how many attempts and at what interval they should take place by filling in the appropriate edit fields.
  13. Select the Audit button in order to enable the reporting of failures.
  14. Push the OK button.

Prerequisites:
The privilege 'Browsing the telecommand options' must be enabled to view the current command history archives settings and the 'Changing the telecommand options' privilege is required to modify them.
