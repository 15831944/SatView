TC Procedures Monitor Execution Options Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Procedures Monitor controls the execution of telecommand procedures. As there are many ways how to and when to trigger the execution of a telecommand procedure, this dialog box allows the specification of all execution related issues.

How do I...

Specify the Execution Triggers for Telecommand Procedures

  1. Open the TC Procedures Monitor Settings Dialog Box if it is not open yet:
    Activate the TC Procedures Monitor and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Contents page.
  3. Choose the telecommand procedure those execution triggers should be changed in the contents list box.
  4. Make sure that the Command Source check box is not selected and push the Execution... button.
    The TC Procedures Monitor Execution Options Dialog Box will occur.
  5. Specify the execution trigger:

    Trigger

    Description

    Immediately

    The telecommand procedure is executed once immediately after specifying this trigger type i.e. after all dialog box are closed.

    Always after start-up

    Starts the execution of the telecommand procedure always after start-up of the application.

    Daily at t

    Initiates the execution of the telecommand procedure every day at the time t.

    At t

    Runs the telecommand procedure at time t.

    At t and repeated every n seconds

    Runs the telecommand procedure at time t and repeats the execution every n seconds

    Optional:

    • Stopped after i times
    • Stopped at t

     

    • Stops execution after i runs
    • Stops at time t
      Note:
      A telecommand procedure running at time t will be completed.

    At event x

    A signaled telemetry event x triggers the execution of the telecommand procedure.

    Note:
    Mark the Allow manual execution check box in order to allow the manual execution of the telecommand procedure. A selected Disabled check box prevents its execution even if a trigger is currently enabled.

  6. Specify the execution mode to be applied:

    Mode

    Description

    Automatic

    The telecommand procedure executes automatically.
    Note:
    If the option Allow manual breaks is enabled the execution can be suspended (or aborted) manually on request. It also allows, when enabled, to set breakpoints associated to any selected telecommand procedure step inside the flowchart panes or to any source code line.
    See also:

    • Suspending the execution of a telecommand procedure
    • Executing a telecommand procedure manually
    • Aborting the execution of a telecommand procedure

    Single-step

    Requires an acknowledgment for each step executed.

    External actions

    Requires an acknowledgment for the steps performing the following tasks:

    • Setting arguments, variables and telecommand parameters
    • Issuing telecommands

    Commands

    Requires an acknowledgment for each step issuing a telecommand.
  7. Push all necessary OK buttons.

Prerequisites:
The privileges 'Changing the contents of the telecommand procedures monitor' or 'Performing privileged tasks on the telecommand procedures monitor' must be enabled to modify execution options.
