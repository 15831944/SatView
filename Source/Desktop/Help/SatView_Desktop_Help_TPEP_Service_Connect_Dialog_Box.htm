TPEP Connection Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Obtaining telemetry data via a TPEP connection may be useful when the client is not a SatView™ workstation or data needs to be sent over low speed transmission lines i.e. phone lines. This dialog box provides a simple way to connect to a TPEP server and to specify the properties of that connection.

How do I...

Connect to a TPEP Server

  1. Open the TPEP Connection Dialog Box if it is not open yet:
    Push the Connection button inside the Telemetry Parameter Exchange Protocol panel of the Communication ribbon bar category.
  2. Type or select the TPEP server foreseen to deliver the telemetry data in the Server combo box.
  3. Mark the View Data check box if a TPEP display showing the telemetry data being exchanged between client and server should be opened automatically.
  4. Select either the Packets or Parameters radio button in the Properties group box depending on the kind of telemetry data requested:

    Requesting telemetry packet data via a TPEP connection

    TPEP connections can return telemetry source packet related data in either an unprocessed format or sorted by telemetry parameters.

    1. Push the Add button to make the TPEP Packets Dialog Box appear or select a predefined connection profile in the Tag combo box of the Profiles group box.
      If the delivery request is constructed with the TPEP Packets Dialog Box select the telemetry packet to be delivered in the Name combo box and push the Add button. Repeat this step as many times as telemetry packets are requested and push the OK button.
      Use the Remove button to delete a previously specified member from the delivery list or push the Remove All button to clear all of them.
    2. Push the OK button.

    Note:
    It is not possible to remove telemetry packets from a TPEP connection that is already in use. In order to modify an open connection close it first and then create a new one with a modified delivery list.

    Requesting telemetry parameter values via a TPEP connection

    TPEP connections can return telemetry parameter values in either an unprocessed format or sorted by telemetry parameters.

    1. Push the Add button to make the TPEP Parameters Dialog Box appear or select a predefined connection profile in the Tag combo box of the Profiles group box.
      If the delivery request is constructed with the TPEP Parameters Dialog Box select the telemetry parameters to be delivered in the Name combo box and push the Add button. Repeat this step as many times as telemetry parameters are requested and push the OK button.
      Use the Remove button to delete a previously specified member from the delivery list or push the Remove All button to clear all of them.
    2. Push the OK button.

    Note:
    It is not possible to remove telemetry parameters from a TPEP connection that is already in use. In order to modify an open connection close it first and then create a new one with a modified delivery list.
  5. Specify the remaining TPEP connection properties by checking or selecting the appropriate controls:

    Property

    Values

    Description

    Link Type

    Hot

    Any changing or updating telemetry packet or parameter on the delivery list is constantly reported.

     

    Cold

    Only a snapshot of the requested telemetry packet(s) or parameter(s) is delivered.

    Data Type

    Raw

    The data of the requested telemetry packet(s) is delivered in an unprocessed (raw) format.
    This option is available for telemetry packets only.

     

    PIE1...n

    A predefined set of information (PIE1...n) is delivered for each parameter within a telemetry packet or for each parameter.
    Consult the document Definition of the Telemetry Parameter Exchange Protocol (TPEP) for a description about PIEs.

    Data Mode

    Real-time

    Delivers telemetry data in either real-time or historical mode.

     

    Historical

    When historical telemetry data is requested a time interval and the retrieval speed must be specified as well.

     

    Random

    Generates and delivers the requested telemetry packet(s) or parameters(s) on a random basis.

    Trigger Mode

    Update

    Any updating telemetry packet or parameter on the delivery list is reported.

     

    Change

    An update only occurs when one of the requested telemetry packets or parameters changed its data or value.
  6. Push the OK button.

Note:
A TPEP delivery list can either contain telemetry packets or parameters but not both.

Prerequisites:
The privilege 'Connecting to a TPEP server' must be enabled in order to connect to a server.
