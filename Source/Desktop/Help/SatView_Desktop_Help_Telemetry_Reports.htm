Telemetry Reports

SUMMARY

Saturday, May 09, 2015
Version 1.0

SatView™ supports a mechanism to automatically generate reports that can be updated based on system alerts, telemetry events, file triggers or a time schedule. Telemetry reports also provide the capability to invoke automation scripts (user defined code) supporting the automation of third-party applications (like Microsoft® Office® applications for example) and provide other powerful features. The generated reports can be mailed or transferred by FTP to any specified user on a daily or weekly basis.

How do I...

Create a new Report Profile

  1. Open the Telemetry Reports Dialog Box if it is not open yet:
    Push the Reports button inside the Reports panel of the Telemetry ribbon bar category.
  2. Enter the name of the new telemetry report profile into the Name edit field.
  3. Mark the Telemetry Events check box if any defined telemetry events should trigger an update of the telemetry report. Select the desired telemetry event inside the associated combo box and push the Add button. Repeat the previous step as may times as telemetry events should trigger an update.
    Use the Remove button in order to delete one of the listed telemetry events or push the Remove All button to delete all of them.
  4. Mark the System Alerts check box if any of the pre-defined system alerts should cause an update of the telemetry report. Select the desired system alert inside the associated combo box and push the Add button. Repeat the previous step as may times as system alerts should trigger an update.
    Use the Remove button in order to delete one of the listed system alerts or push the Remove All button to delete all of them.
  5. Push the File Trigger... button if a telemetry report update should be triggered by the creation, modification or deletion of one or more files in a single or multiple directories. The Telemetry Report File Triggers Dialog Box will pop-up. Specify the directories to be monitored and what type of action should be triggering a telemetry report update. If only specific files are of interest (i.e. not all of the directory) they can also be specified (wildcard characters are allowed). Even more, a different action can be specified for each of them (if required).
    Note:
    There is a potential danger for infinite loops in cases where the telemetry report executes an automation script which itself triggers the same report by creating, modifying or deleting a file in a directory being monitored by the same report. However, if such a setup is required, the automation script must contain code to break the potential loop.
  6. Push the Schedule... button in order to trigger the telemetry report by time. The Schedule Telemetry Reports Dialog Box will appear. Specify when and how many times the telemetry report should be executed by checking the corresponding radio buttons and entering the requested time.
    Note:
    This schedule option can be combined with the the triggers explained before.
    A scheduled execution can be removed by un-checking all schedule options in the Schedule Telemetry Reports Dialog Box or by using the Abort Telemetry Reports Dialog Box. Otherwise, a scheduled (non periodical) trigger is deleted automatically when it has fired.
  7. Select an automation script in the Script combo box which should run each time the telemetry report is updated.
    Note:
    Specifying 'None' as automation script performs a standard behavior by just appending the name of the telemetry event or system alert (including the time) to the output file associated with the report.
  8. Enter the input file name(s) into the Input File edit field if required.
    Note:
    This option is available only if a non default automation script is attached to the report. The files specified as input files are handed over to the automation script for further processing.
  9. Enter the output file name into the Output File edit field. This file will be used to contain the contents of the telemetry report.
    Note:
    Mark the Keep All check box in order to keep all generated telemetry reports. If this option is not checked, the report is deleted whenever it has been sent to the registered users or the daily/weekly notification period expires.
  10. Push the < button in order to add any e-mail recipients to this telemetry report profile. An additional button labeled with FTP... is available to specify a FTP location to receive the report.
    Use the > button to remove any specified recipients.
  11. Mark the All n days at t or Weekly at t check box and specify a time t when the telemetry report should be sent to the registered users. For the daily option it is possible to specify a number of days n in addition to the time.
    Note:
    It is also possible to specify neither one of both options. This will prevent that the telemetry report is sent to any registered recipients.
  12. If necessary, a telemetry report can be set inactive by marking the Disabled check box.
  13. Push the Add button.
    Note:
    A button labeled with Abort... is available to abort a currently executing telemetry report or to remove a queued one (see Abort Telemetry Reports Dialog Box).
  14. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry reports' must be enabled to create, modify or delete a telemetry report profile.

Modify a Report Profile

  1. Open the Telemetry Reports Dialog Box if it is not open yet:
    Push the Reports button inside the Reports panel of the Telemetry ribbon bar category.
  2. Select the name of the telemetry report profile to be modified in the Name combo box and perform the required changes.
  3. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry reports' must be enabled to create, modify or delete a telemetry report profile.

Delete a Report Profile

  1. Open the Telemetry Reports Dialog Box if it is not open yet:
    Push the Reports button inside the Reports panel of the Telemetry ribbon bar category.
  2. Select the name of the telemetry report profile to be deleted in the Name combo box.
  3. Push the Remove button. Use the Remove All button in order to delete all telemetry report profiles at once.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry reports' must be enabled to create, modify or delete a telemetry report profile.

Import Report Profiles

  1. Open the Telemetry Reports Import Dialog Box if it is not open yet:
    Push the Import... button inside the Reports panel of the Telemetry ribbon bar category.
  2. Enter the name of the file that contains all telemetry report profiles to be imported into the Filename edit field.
    Use the Browse... button in order to search for an appropriate source file if necessary.
  3. Push the Inspect button in order to list all telemetry report profiles that are contained in the specified file.
    Note:
    The Inspect button is available only when a file name is specified. With an empty Filename edit field the Browse... button is visible instead.
  4. In the Directory combo box select a telemetry report profile that should be imported.
  5. Push the Add button. Repeat the previous two steps if more than one telemetry report profile should be imported or use the Add All button in order to import all available ones.
    Use the Remove or Remove All button in order to remove one or several telemetry report profiles from the importation list.
    Note:
    Telemetry report profiles can be imported in one step from more than one source file by entering a different file name into the Filename edit field and following the steps above.
  6. Push the OK button.

Prerequisites:
The privilege 'Importing telemetry reports' must be enabled to import telemetry report profiles.

Export Report Profiles

  1. Open the Telemetry Reports Export Dialog Box if it is not open yet:
    Push the Export... button inside the Reports panel of the Telemetry ribbon bar category.
  2. In the Directory combo box select a telemetry report profile that should be exported.
  3. Push the Add button. Repeat the previous steps if more than one telemetry report profile should be exported or use the Add All button in order to export all available ones.
    Use the Remove or Remove All button in order to remove one or several telemetry report profiles from the exportation list.
  4. Enter the name of the file that should contain all exported telemetry report profiles into the Filename edit field.
    Use the Browse... button in order to search for an appropriate destination file if necessary.
  5. Push the OK button.

Prerequisites:
The privilege 'Exporting telemetry reports' must be enabled to export telemetry report profiles.

Generate a historical Report

Telemetry reports can also be generated from historical telemetry data. This is a powerful feature helping to create reports at any time convenient provided that archived telemetry data is available for the desired time period.

  1. Open the Historical Telemetry Reports Dialog Box if it is not open yet:
    Push the Historical Reports button inside the Reports panel of the Telemetry ribbon bar category.
  2. Enter or select the name of the historical telemetry report profile to be created or modified in the Name combo box.
  3. Select the telemetry report to be generated in the Report combo box.
  4. Enter the historical telemetry data file to be used for the generation of the telemetry report into the Source edit field.
    Note:
    The associated combo box contains a list of predefined history files that can be used as a source.

  5. Specify the time range the telemetry report should cover.

    1. Select the Range page
    2. Specify the time range the telemetry report should cover by choosing one of the following options:

      Option

      Description

      From t1 to t2

      All telemetry data available between the specified start time t1 and stop time t2 is included in the telemetry report.
      Note:
      Depending on the kind of telemetry data standard supported it is not always possible to guarantee that all data within the specified time range is included. Therefore, its recommended to declare a slightly larger period than required.

      Daily from t1 to t2

      The time interval for the telemetry report is derived from a daily period always starting at the specified time t1 and stopping at t2.
      Note:
      Depending on the kind of telemetry data standard supported it is not always possible to guarantee that all data within the specified time range is included. Therefore, its recommended to always declare a slightly larger period than required.

      Periodically: All available time

      Includes periodical time intervals in an intermittent way into a telemetry report.

      Periodically: The last n minutes of the interval

      Includes telemetry data at a specified periodical interval into a telemetry report but only the last n minutes of every interval.

      Triggered: The last n minutes before/after the event

      Includes the last n minutes before and/or after the occurrence of a specified telemetry event into a telemetry report.

      Use an interval of n minutes

      Directs the report generation process to step ahead in intervals of n minutes instead of progressing on a continuous basis.
      Note:
      Depending on the kind of telemetry data standard supported the resulting steps may be smaller or larger than the specified interval.

  6. Specify the execution time of the telemetry report profile.

    1. Select the Execution page
    2. Choose one of the available execution modes by checking the appropriate radio button:

      Mode

      Description

      Immediately

      The historical telemetry report profile is executed immediately.

      Once at t

      The historical telemetry report profile starts executing at time t.

      Daily at t

      The historical telemetry report profile is executed daily at time t.

      Starting at t1 with an interval of n minutes and stopping at t2

      The execution of the historical telemetry report profile takes place periodically with an interval of n minutes, starting at time t1 and stopping at t2.

      At event x

      Executes the historical telemetry report profile whenever the event x occurs.

      Note:
      The number of available execution modes depends on the type of time range chosen.

  7. Specify the input files of the telemetry report profile (optional).

    1. Select the Input page
    2. Enter the name of the input files required for the telemetry report generation (if any) into the Files edit field.
      Use the Browse... button to search for any files.
      Note:
      • Multiple files must be separated by a semicolon.
      • Wildcards (e.g. input*.*) as part of the file name are allowed. In this case an automatic search for the files will be initiated.
    3. Specify a directory for the input files by typing it into the Directory edit field.
      Use the Browse... button to look for a folder.
      Note:
      The temporary directory is used as target location for any files that need to be downloaded if no directory is specified.
    4. Mark the Look at check box if the input files have to be downloaded from a remote location via FTP and fill the necessary information like FTP address ('ftp://domain/directory' where 'domain' identifies a domain name or IP address and 'directory' an existing directory), user name, password and eventually port number into the Username, Password or Use Port edit fields.
    5. Mark the Keep Files check box if the input files should not be deleted after the generation of the telemetry report.

  8. Specify the output file name and the recipients of the telemetry report (optional).

    1. Select the Output page
    2. Enter the name of the output file into the Filename edit field.
    3. Mark the Keep File check box if the generated output file should not be deleted after it has been sent to any recipients.
    4. Push the < button in order to add any e-mail recipients to this telemetry report profile. An additional button labeled with FTP... is available to specify a FTP location to receive the telemetry report.
      Use the > button to remove any specified recipients.
    5. Mark the Keep Files check box if the input files should not be deleted after the generation of the telemetry report.
  9. Push the Save button to save the new or modified historical telemetry report profile.
  10. Push the Submit button in order to execute or schedule the historical telemetry report profile depending on the chosen execution mode.
    Note:
    Historical telemetry report profiles triggered by telemetry events do not need to be submitted.
  11. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry reports' must be enabled to create, modify or delete a historical telemetry report profile.


Related Topics:

Abort executing Reports

  1. Open the Telemetry Reports Abort Dialog Box if it is not open yet:
    Push the Reports button inside the Reports panel of the Telemetry ribbon bar category and push the Abort... button in the dialog box.
  2. Inside the List list box all telemetry report jobs are enumerated that are currently executing or waiting to be executed. Select only those jobs that should be aborted.
    Note:
    The lower Triggered List list view shows all telemetry report profiles currently configured to be triggered by a telemetry event. The list is available for informational purposes only.
  3. Push the OK button.

Note:
In some cases the abortion of telemetry report jobs has no immediate effect. This happens in situations where a re-schedule of the execution queue happened while this dialog box was open.

Abort executing historical Reports

  1. Open the Telemetry Reports Abort Dialog Box if it is not open yet:
    Push the Historical Reports button inside the Reports panel of the Telemetry ribbon bar category and push the Abort... button in the dialog.
  2. Inside the List list box all telemetry report jobs are enumerated that are currently executing or waiting to be executed. Select only those jobs that should be aborted.
    Note:
    The lower Triggered List list view shows all telemetry report profiles currently configured to be triggered by a telemetry event. The list is available for informational purposes only.
  3. Push the OK button.

Note:
In some cases the abortion of telemetry report jobs has no immediate effect. This happens in situations where a re-schedule of the execution queue happened while this dialog box was open.
