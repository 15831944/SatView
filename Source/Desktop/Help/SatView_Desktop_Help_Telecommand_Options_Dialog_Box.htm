Telecommand Options Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

The telecommand service is essential for the correct generation, release and verification of telecommands. Its major component is the central command handler coordinating the release of telecommands issued from the various (distributed) command sources. The command handler can be controlled from any client (provided that the necessary privilege is active).

How do I...

Configure the Command Handler

The central command handler coordinates the release of telecommands issued from the various (distributed) command sources. In addition, it is also possible to manage the global telecommand release modes and constraints. The command handler can be controlled from any client (provided that the necessary privilege is active).

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the Configuration page.
  3. Read more about:

    Choosing the Options

    1. Select the host for the primary command handler and, if required, for the secondary (backup) handler by checking the corresponding check box and by entering the appropriate names.
      Note:
      If no names are supplied, one unique host (with the 'Allow this workstation to be a command handler' option enabled) will be selected automatically.
    2. Select the appropriate options:

      Option

      Description

      Allow this workstation to be a command handler

      Confirms that the current workstation can be or get a command handler (if required).

      Use any available command handler

      Searches for an available command handler first before getting one itself (when required).

      Ensure a unique command handler

      Makes sure that only one workstation can be the primary command handler.

      Always allow exit

      Shutting down the command handler is possible even when there are still telecommands to be verified.
      Note:
      It is never possible (even when this option is enabled) to exit as long as there are still telecommands in the release queue.

    Configuring the Network Settings

    1. Open the Command Handler Settings Dialog Box: Push the More... button.
    2. Select the port to be used by the command handler through selecting the appropriate number in the Port spin box.
      Note:
      It is important that all workstations authorized to issue telecommands share the same port number.
    3. Adjust, if required, the buffer size to be allocated for the communication between the command handler and client workstations with the Buffer spin box. The same applies for the timeout allowed for the network which can be specified with the Timeout spin box.
    4. Specify the workstations allowed to log-in to the command handler by adding their address properties into the Clients list box. This can be achieved by pushing the Add button which opens the Command Handler Client Addresses Dialog Box. The name and IP address of the client workstations can then be entered.
      Note:
      By pushing the Allow access for any non-listed client check box grants non-listed clients the permission to log-in.
    5. For each listed client it is possible to specify how the access for this client can take place:

      Option

      Description

      None

      Allows unrestricted access for the selected client.

      Ban Client

      Denies access for a client.

      From t1 to t2

      Allows temporary access between t1 and t2 only.

      Note:
      The Audit check box can be marked to generate an event whenever a banned client tries to log-in.

    6. Push the OK button.

    Using the Global Telecommand Release Modes

    • Choose the appropriate global telecommand release modes:

      Mode

      Description

      Master Manual Mode

      Prevents all command sources from issuing any telecommands except for the TC Manual Stack hosted on the workstation specified in the associated combo box.

      Release Wait Mode

      Forces all command sources to apply the release wait mode which allows the release of telecommands within the specified (default) time window. If this mode is not enabled, a telecommand release is considered as failed as soon as the release time is encountered.

      Automatic Rejection Mode

      This mode automatically removes a telecommand that failed to be released from the TC Manual Stack or TC Automatic Stack (schedule pane).

      Disable telecommand release

      Prevents the issue of telecommands from any command source.

      Note:
      The change of any of these modes applies to all command sources. However, it is possible to overwrite these settings for a certain command source separately at a later time.

    Using the Global Telecommand Release Constraints

    1. Choose the appropriate global telecommand release constraints:

      Constraint

      Description

      Override Pre-Transmission Validation (PTV)

      Forces any pre-transmission validation checks to result in a positive ('Go') result even when some or all conditions are not met.

      Override Pre-Execution Verification (PEV)

      Forces any pre-execution verification checks to result in a positive ('Go') result.

      Override Execution Verification (CEV)

      Forces any execution verification checks to result in a successful ('Passed') result.

      Disable PTV

      Does not perform any pre-transmission validation checks (implicit 'Go').

      Disable PEV

      Does not perform any pre-execution verification checks (implicit 'Go').

      Disable CEV

      Does not perform any execution verification checks.

      Note:
      The change of any of these constraints applies to all command sources. However, it is possible to overwrite these settings for a certain command source separately at a later time.

    2. Adjust the verification uncertainty interval in order to extend the time span allowed for the telecommand execution verification by entering the correct number into the Verification uncertainty interval spin box.
    3. In order to activate the interlock mechanism for all command sources check the Enable Interlocks check box.

    Changing Global Time-out Intervals

    1. Adjust the maximum (future) time allowed for on-board execution times of telecommands in the Maximum future time period for telecommand execution requests spin box.
    2. Enter the latest time allowed for any parameter of a telecommand into the For time checks spin box.
    3. Modify the default release time window by changing the value in the Default release time window interval spin box.

    Note:
    The time-out intervals specified apply to all command sources. The default release time window interval, however, can be overwritten for each command source locally (at a later time).

    Showing or hiding the CLCW Panel

    • Mark the Show CLCW Panel check box if the Command Link Control Word panel should be displayed.

    Note:
    The CLCW panel displays the content of the command link control word and allows to correlate the TC Transfer Frame Sequence Number with the next value expected by the spacecraft.
  4. Push the OK button.

Configure & Control Command Sources

Any (client) workstation that has the required privileges can monitor and control all other (remote) command sources. Changing the operating modes or release constraints of a command source overwrites the corresponding global settings (only if applied later).

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the Command Sources page.
  3. Read more about:

    Using Telecommand Release Modes

    1. Select the computer hosting the command source to be re-configured inside the Computer list box.
    2. Choose the type of command source desired by checking either of these radio buttons: TC Manual Stack, TC Automatic Stack or TC Procedures Monitor.
    3. Choose the appropriate telecommand release modes for the selected command source:

      Mode

      Description

      Master Manual Mode

      Prevents all command sources from issuing any telecommands except for the TC Manual Stack hosted by the selected computer.

      Release Wait Mode

      Forces the selected command source to apply the release wait mode which allows the release of telecommands within the specified (default) time window. If this mode is not enabled, a telecommand release is considered as failed as soon as the release time is encountered.

      Automatic Rejection Mode

      This mode automatically removes a telecommand that failed to be released from the selected TC Manual Stack or TC Automatic Stack (schedule pane).

      Disabled

      Prevents the issue of telecommands from the selected command source.

      Note:
      Some command sources do not support all of these telecommand release modes.

    Using Telecommand Release Constraints

    1. Select the computer hosting the command source to be re-configured inside the Computer list box.
    2. Choose the type of command source desired by checking either of these radio buttons: TC Manual Stack, TC Automatic Stack or TC Procedures Monitor.
    3. Choose the appropriate telecommand release constraints for the selected command source:

      Constraint

      Description

      Override Pre-Transmission Validation (PTV)

      Forces any pre-transmission validation checks to result in a positive ('Go') result even when some or all conditions are not met.

      Override Pre-Execution Verification (PEV)

      Forces any pre-execution verification checks to result in a positive ('Go') result.

      Override Execution Verification (CEV)

      Forces any execution verification checks to result in a successful ('Passed') result.

      Disable PTV

      Does not perform any pre-transmission validation checks (implicit 'Go').

      Disable PEV

      Does not perform any pre-execution verification checks (implicit 'Go').

      Disable CEV

      Does not perform any execution verification checks.

      Note:
      Some command sources do not support all of these telecommand release constraints.

    4. In order to activate the interlock mechanism for the selected command source check the Enable local Interlocks check box.
    5. For the some command sources it is possible to automatically enable/disable the release of telecommands targeting a certain sub-system on-board of the spacecraft depending on telemetry events.
      This can be achieved by pushing the Sub-systems... button which opens the Command Source Sub-systems Dialog Box:

      Manually controlling the release of telecommands for a certain sub-system

      1. Select the on-board sub-system for which the release of telecommands should be allowed or denied inside the Sub-systems list box.
      2. Choose the Enabled check box to allow the release of telecommands or Disabled to deny it.
        Note:
        In order to force a constant setting (i.e. independent from any telemetry data) no events must be attached.
      3. Push the OK button.

      Controlling the release of telecommands for a specified sub-system by telemetry events

      1. Select the on-board sub-system for which the release of telecommands should be allowed or denied inside the Sub-systems list box.
      2. Push the Enabled check box and select the telemetry event enabling the release of telecommands in the upper combo box and/or choose the event disabling the release in the lower combo box.
        Note:
        If no telemetry event is attached to the enabled state, the release of telecommands is always allowed except for situations where the event assigned to the disabled state is signaled. In the other case where an event is attached to the enabled state only, the release of telecommands is allowed solely when this event is signaled.
      3. Push the OK button.

    Changing Time-out Intervals

    1. Select the computer hosting the command source to be re-configured inside the Computer list box.
    2. Choose the type of command source desired by checking either of these radio buttons: TC Manual Stack, TC Automatic Stack or TC Procedures Monitor.
    3. The default release time window interval for the selected command source can be changed by increasing or decreasing the number inside the Default release time window interval spin box.
    4. The dispatch time interval i.e. the period between dispatching a telecommand and the time when it switches back to non-dispatched state automatically (when not released) can be changed in the Dispatch time window interval spin box.
  4. Push the OK button.

Manage Interlocks

Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the Interlocks page.
  3. Read more about:

    Creating, modifying or deleting Interlocks

    1. Push the Edit... button. The Telecommand Interlocks Dialog Box will appear.
    2. Enter a unique name into the Name edit field to create a new interlock or select an existing one inside the associated combo box for modification or deletion.
    3. Specify or change the scope of the interlock:

      Scope

      Description

      Global (all command sources)

      Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

      Local (same command source only)

      Blocks the local command source as long as the interlock is in a pending or failed state.
      Note:
      Enable the No wait state option if the interlock should not block when in a pending state.

      Local on specified subsystem

      Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
      Note:
      Enable the No wait state option if the interlock should not block when in a pending state.

      Local on specified telecommands

      Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
    4. Select or change the command source to host the interlock by selecting the computer in the Computer combo box and by pushing either TC Manual Stack, TC Automatic Stack or TC Procedures Monitor radio button.
    5. Choose the telecommand verification stage to affect the resulting state of the interlock by marking either the Reception (at ground station), Transmission, Acceptance, Start, Progress or Completion check box.

    7. Push the OK button.

    Only those interlocks that are in an idle state can be modified or deleted. If an interlock is in a success or failure state it has to be reset first. Pending interlocks can neither be modified nor deleted.

    Resetting Interlocks

    1. Select the interlock to be reset in the Name combo box.
    2. Push the Reset button.

    Note:
    An interlock can only be reset when it is in a success or failure state.
  4. Push the OK button.

Manage the Command Release Queue

All telecommands issued from a command source with a non-immediate release time are queued in the central command handler. This queue can be manipulated from any workstation provided the required privilege is granted.

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the Queue page.
  3. Select the telecommands to be removed in the list box and push the Remove button. The Remove All button empties the queue completely.
    Note:
    Deleting a telecommand that is part of a sequence removes the complete (or remaining, non-released part of that) sequence.
  4. Push the OK button.

Configure the Command History Archives

Telecommands released by the central command handler or those failed to be transmitted can be archived for later replay. Many archive options are available, including automatic backups, that simplify the command history management.

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the History page.
  3. Read more about:

    Specifying the primary/backup Command History File

    1. Enter a file name into the Telecommand history file edit field. Use the Browse... button to search for an existing file.
    2. Type the file name of the backup command history file into the Backup history file edit field if a backup file is required. A backup file is simply a duplication of the primary one i.e. archiving takes place on both files simultaneously.

    3. Configure the Files

      1. Push the Options... button. The Telecommand Archive Options Dialog Box opens.
        Note:
        A name for either the primary or backup command archive file must be supplied before any options can be modified.
      2. Choose the file to which the options should be applied to in the Filename combo box.
      3. For each command archive file there is a possibility to attach another one that is updated only when the first fails. This can be achieved by marking the Use filename on failures check box and entering a valid file name into the associated edit field.
        Note:
        A command history file can temporarily be disabled (i.e. no archiving takes place) by marking the Disabled check box.
      4. Choose one of the available archiving policies by checking the corresponding radio button inside the Properties group box:

        Option

        Description

        Cyclic

        Archiving is done on a cyclic basis. Whenever the specified file size is reached or no space on the storage media is available any more the file is incrementally overwritten from its beginning.

        Noncyclic

        Archiving is performed until the specified file size is reached or no space is available any more on the storage media.
      5. Specify the size of the file by choosing on of these options:

        Option

        Description

        Unlimited

        All the available space on the storage media is used up for archiving.

        Restricted to n MB

        Archiving is performed until the specified file size of n MB is reached.
      6. Mark the Use Compression check box if a file based compression should take place during archiving.
        Note:
        This option is only effective on storage medias supporting file compression.
      7. Choose one of the available backup options by checking the corresponding radio button inside the Backup group box:

        Type

        Description

        Simple

        The backup file has the same name as the archive file: 'Backup of filename.TCH'.
        Note:
        If more than one backup file is generated with this type the name is: 'Backup (n) of filename.TCH'.

        Timetagged

        A part of the backup file name is the time key of the earliest telemetry data contained in the file: 'YYYY.MM.DD_hh.mm.ss.TCH'.

        Enumerated

        Enumerates the backup files by using the same name as the archive file: 'Filename_n.TCH'.
      8. Specify the directory where the backup files should be moved to by typing its path into the Directory edit field.
      9. Select one or more of the triggers determining the time when a backup should take place by marking the corresponding check boxes:

        Trigger

        Description

        When file reached its specified size

        Whenever the archive file reaches its predefined size a backup is performed.
        Note:
        The archive file is always deleted after a backup has taken place.

        When file size is at least n MB

        Performs a backup when the archive file reached a specified size of n MB.
        Note:
        This trigger is not available for cyclic archive files.
      10. Mark the Perform Backups check box if backups of the archive file should be done.
      11. Choose one or more of the behavior options by selecting the corresponding check boxes:

        Option

        Description

        No backups between t1 and t2

        Any attempt to perform a backup of this archive file is delayed to t < t1 or t > t2.

        No backups on non-empty storage medias

        Prevents a backup of this archive file as long as a non-empty storage media is inserted. The backup is performed later when an empty storage media is inserted.
      12. Mark the On Failures retry n times check box if n attempts should be tried before discarding the file and eventually issue a warning message. When enabling this option specify how many attempts and at what interval they should take place by filling in the appropriate edit fields.
      13. Select the Audit button in order to enable the reporting of failures.
      14. Push the OK button.
    4. Enable the command history archiving by marking the check boxes to the left of the file name edit fields. Unselecting those check boxes pauses the attached archive file.
    5. Push the OK button.

    14. Push the OK button.

    Deleting the primary/backup Command History File

    • Mark either the Delete Telecommand history file check box in order to delete the primary command history file or the Delete Backup history file check box to erase the redundant one from the storage media.
      Note:
      This option is available only when a file name exists. Trying to delete a non-existing file, however, does not result in an error message.

    Specifying the Command History Files for Retrievals

    1. Type the file name into the edit field of the Files combo box. Use the Browse... button to search for an existing retrieval file to be added.
      The Inspect button can be pushed to verify if a certain file name represents a valid retrieval file. All information about the inspected file is displayed inside the Source Information group box.
      Note:
      By marking the Default File check box the currently selected file will be considered as the default one i.e. any retrieval activities will take place on that file if not specified differently.
    2. Push the Add button in order to add it to the list of retrieval files.
      Repeat the previous steps if more than one file should be added or use the Remove button to exclude the selected retrieval file from being part of the command history files. The Remove All button empties the list of command history files available for retrievals.
  4. Push the OK button.

Configure the Command Handler Logging Options

Configuration changes for the central command handler can be logged in order to view these activities at a later time. Logging capabilities exists also for each of the command sources and the resulting files can be merged all together, if desired, to reduce the total number of logs to maintain.

  1. Open the Telecommand Options Dialog Box if it is not open yet:
    Push the Options button inside the Configuration panel of the Telecommand ribbon bar category.
  2. Switch to the Logging page.
  3. Read more about:

    Configuring the Options

    1. Push the Options... button. The Telecommand Logging Options Dialog Box will appear.
    2. Type the name for the log file into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Note:
      The file name can be identical to those of other (remote) command handlers or even command sources. This may be interesting if only one single log file should be generated.
    3. Check one of the available file policy options:

      Option

      Description

      Overwrite logs as needed

      Enables the cyclic overwrite mode.
      Note:
      As soon as the log file reaches the specified size or no more space is available on the storage media new logs overwrite the oldest ones.

      Overwrite logs older than n days

      Enables the cyclic overwrite mode.
      Note:
      Logs are overwritten by new ones as soon as they are older than n days. This can happen even before the specified file size is reached.

      Do not overwrite any logs

      Disables the cyclic overwrite mode.
      Note:
      Stops logging when either the specified size is reached or no more space is available on the storage media.
      Optional:
      • Mark the Disabled check box in order to prevent the log file from being updated.
    4. Check one of the available file size options:

      Option

      Description

      Unlimited

      Archives logs as long as space is available on the storage media.

      Limited to n KBytes

      Limits the file size to n KB.
      Optional:
      • Mark the Reset check box in order to reset the log file.
    5. Push the Retrievals... button which opens the Telecommand Logging Retrieval Settings Dialog Box and type the name of the log file to be used for retrievals into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Push the OK button.
      Note:
      The file name is usually identical to the one specified for logging but can also be identical to those of other (remote) command handlers or even command sources. This may be useful if only one single file is used for the logging.
    6. Push the OK button.

    Viewing Real-time Log Entries

    1. Check the Realtime radio button.
    2. Mark the Reload the last n entries check box, enter the number n into the associated edit field and push the Retrieve button in order to immediately retrieve the last n log entries.
      Note:
      By enabling this option the last n log entries will always be listed whenever the dialog box is opened.

    Retrieving Log Entries

    1. Check the Historical radio button and enter the start time for the retrieval into the associated edit field.
    2. Specify the stop condition by checking either:
      • Stop at t and enter the desired stop time t
      • Limit to n entries and enter the number n of entries to retrieve
      • Retrieve up to the end of the log file radio button
    3. Push the Retrieve button.

    Redirecting Log Entries

    • Mark the Show Event check box in order to redirect each log entry as an event to the Global Eventbox.
  4. Push the OK button.

Prerequisites:
The privilege 'Browsing the telecommand options' must be enabled to view the existing command handler configuration and the 'Changing the telecommand options' privilege is required to modify it.
