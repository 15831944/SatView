Telemetry Data Extractions Abort Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ performs archiving of telemetry data on the basis of normal files. Multiple files can be specified and each of those files can be configured separately with respect to its properties and behavior. Furthermore, some effort has been made concerning automatic backups and to assure that no telemetry data is lost in case of failures. This dialog box allows the abortion of one or more (queued) telemetry data extraction profiles.

How do I...

Abort executing Telemetry Data Extraction Profiles

  1. Open the Telemetry Data Extractions Abort Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category, select the Extractions page in the dialog box and push the Abort... button.
  2. Inside the List list box all telemetry data extraction jobs are enumerated that are currently executing or waiting to be executed. Select only those jobs that should be aborted.
    Note:
    The lower Triggered List list view shows all telemetry data extraction profiles currently configured to be triggered by a telemetry event. The list is available for informational purposes only.
  3. Push the OK button.

Note:
In some cases the abortion of telemetry data extraction jobs has no immediate effect. This happens in situations where a re-schedule of the execution queue happened while this dialog box was open.
