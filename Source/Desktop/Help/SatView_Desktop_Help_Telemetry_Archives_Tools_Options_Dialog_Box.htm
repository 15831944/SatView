Telemetry Data Tools Options Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

History files generated from one or multiple source files can have different formats in SatView™; in most cases those files are used as a normal history file but it is also possible to create them in a human readable way i.e. as an ASCII dump.

How do I...

Specify History File Format Options

  1. Open the Telemetry Data Tools Options Dialog Box if it is not open yet:
    Push the Settings button inside the Archives panel of the Telemetry ribbon bar category, select the Tools page in the dialog box, choose the Raw format as the selected one and push the Options... button.
  2. Choose either the ASCII or Binary radio button depending on the requested encoding for the resulting file(s).
  3. Inside the Format group box choose the requested radix and formatting options by checking the relevant radio buttons.
    In addition, select the Unicode check box if the output file(s) should use the unicode character set.
    Note:
    The format options are available only when ASCII encoding is requested.
  4. Extracted telemetry units can be separated by a carriage-return, an end-of-line character or a combination of both by choosing the appropriate None, CR, EOL or CR+EOL radio buttons.
    Note:
    The default for binary encoding is to insert no separator, for ASCII it is a carriage-return followed by an end-of-line sign.
  5. Sometimes it might be useful to create multiple output files i.e. one for each type of telemetry unit. This can be achieved by marking the Use a separate file for each telemetry unit type check box.
    If this option is chosen, it is also possible to specify a file prefix via the Use the file prefix check box and an extension by selecting Use the file extension check box and entering the relevant information.
  6. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry data history file extraction facility' must be enabled to create historical telemetry data file containing selected periods of data.
