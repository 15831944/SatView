Telemetry Service

SatView™ can be made compatible with a variety of telemetry data provider products. By implementing the concept of virtual devices where for each physical device to be connected to needs a corresponding software driver, all device dependencies can be encapsulated in an elegant way.
In order to provide a maximum availability of telemetry data, more than one device can be added to the list of potential telemetry data providers. Whenever SatView™ requires telemetry data, it steps through that list until an enabled device responds. All these devices can be enabled or disabled and scheduled individually. Additional options allow the specification of backup devices.

Memory Dumps Service

SatView™ is able to acquire and process telemetry data dumped from the spacecraft through the Memory Dumps Service. The collected data can (optionally) be further processed by user-defined automation scripts, archived or sent to other users via FTP transfer. The Dumps type of the Hexadecimal Display (HEX) is the first choice to view such memory dumps. Each memory dump to be processed must be registered by an associated profile. The Memory Dump Service is a light-weight solution compared to the TC On-board Memory Images Display which is used to reflect the complete on-board memory status of a spacecraft.

PFLP Service

SatView™ supports a special protocol called Packet Front-End Link Protocol (PFLP) designed to export real-time telemetry data in form of telemetry source packets. It operates on top of TCP/IP and is session based. Each SatView™ workstation can handle an unlimited amount of clients and provides an easy way how to control the availability of the service including client access privileges.

TPEP Service

SatView™ supports a special protocol called Telemetry Parameter Exchange Protocol (TPEP) designed to export real-time and historical telemetry data in various forms. It operates on top of TCP/IP and is session based. Each SatView™ workstation can handle an unlimited amount of clients and provides an easy way how to control the availability of the service including client access privileges.

Archives

SatView™ is able to archive the telemetry data on the basis of files. More than one file can be specified for archiving and together with a variety of options the data can be stored in many different ways. In general, the telemetry data is archived in a raw (unprocessed) form that allows the replay at a later time even with a modified database. However, in order to be compatible with incremental telemetry data streams (for example the CCSDS Packet Telemetry Standard) and to guarantee correct retrievals, the values of all telemetry parameters are embedded into the archive file at a definable interval.
Retrievals are performed through the various displays and can take place at the same time as the real-time processing. Due to the fact that the telemetry data can be distributed over many files, it is possible to determine a default retrieval file and to change to another one for each retrieval request separately.
Furthermore, the implemented backup capabilities simplify the way how to handle long-term archiving. Sometimes it may also be advantageous to extract values for a set of parameters over a specified amount of time. This is a supported feature and particularly useful to process the extracted telemetry data offline, for example, inside a spreadsheet. Extractions are not limited to specified parameter sets but can also include complete portions of history files originating from multiple sources. With this capability, interesting periods of telemetry data can be extracted and distributed as a fully functional history file.

Displays

SatView™ supports various types of telemetry data related displays. They monitor the telemetry parameter values in many ways: Raw, calibrated, pre-processed by user-defined automation scripts, presented in a listed form, graphically or in a mimics layout. Each of those displays can also replay historical telemetry data and is equipped with a rich set of configuration options.

Events

SatView™ supports a mechanism to allow users to be notified in various ways when a certain predefined telemetry event occurs. Such an event can be identical to a telemetry parameter smaller, equal, non-equal or bigger than a constant value or a more complex expression consisting of any thinkable mathematical calculation (comparable to a derived parameter). Any existing audition and notification profile can be attached to such an event.

Reports

SatView™ supports a mechanism to automatically generate reports that can be updated based on system alerts, telemetry events, file triggers or a time schedule. Telemetry reports also provide the capability to invoke automation scripts (user defined code) supporting the automation of third-party applications (like Microsoft® Office® applications for example) and provide other powerful features. The generated reports can be mailed or transferred by FTP to any specified user on a daily or weekly basis. Furthermore, it is possible to generate telemetry reports based on historical telemetry data.

Parameters

SatView™ provides a convenient interface to view the properties of telemetry parameters. Type, nature, coding of a parameter as well as other relevant information is displayed. For constants and derived parameters it is also possible to modify the current value. This is particularly useful for iterative derived parameters which always need an initialization.
