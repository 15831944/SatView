Telemetry Displays

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ supports various types of telemetry data related displays. They monitor the telemetry parameter values in many ways: Raw, calibrated, pre-processed by user-defined automation scripts, presented in a listed form, graphically or in a mimics layout. Each of those displays can also replay historical telemetry data and is equipped with a rich set of configuration options.

Alphanumeric Displays (AND)

An Alphanumeric Display (AND) presents parameters in an alphanumeric format. Its parameters can be arranged in groups with associated labels and are capable to show values in various number formats. Furthermore, the amount of information displayed for each parameter is configurable by choosing a set of predefined columns to be part of the layout.

Graphic Displays (GRD)

A Graphic Display (GRD) presents parameters in a plotted layout. The various chart types available allow the monitoring of parameter values against time or against other parameters and offer a broad range of options.

Hexadecimal Display (HEX)

A Hexadecimal Display (HEX) is capable to monitor raw (unprocessed) telemetry data as well as Memory Dumps. In addition to that, so called Spy Locations extract selected portions of the data for further investigation.

Out-of-Limit Display (OOL)

An Out-of-Limit Display (OOL) lists all parameters that are in an out-of-limit condition. Sophisticated filters allow to reduce the displayed parameters and also provide the capability to specify how selected parameters should be alerted: Auditions and notifications can be linked to individual parameters causing the related action to be executed when they slip into an out-of-limit condition or leave it.

Mimics Display (MMD)

A Mimics Display (MMD) is a set of dynamic graphic objects that are animated by telemetry data. A custom language, the Mimics Description Language (MDL) has been developed to control the mimics objects. Individual areas of the display can be linked with other MMDs allowing a kind of zoom-in functionality.

Parameter Observation Display (POD)

A Parameter Observation Display (POD) presents parameter values or derived expression results in a scrollable, alphanumeric manner. Expressions can be specified 'on-the-fly' and may contain mathematical functions as well as parameter values. In addition, it is possible to define a list of parameters to be added to the display for comparison reasons and a predefined set of statistical functions helps to analyze the resulting parameter values or expression results in more detail.

Telemetry Parameter Exchange Protocol Display (TPEP)

A Telemetry Parameter Exchange Protocol Display (TPEP) displays the message transfer of an existing TPEP connection. It can show the messages as they are exchanged or sorted by parameters. The alert option can automatically watch parameter values and generate a notification on request. The TPEP display is also useful to monitor a selected set of parameters over low speed transmission lines or in a heterogeneous environment.

Eventbox

An Eventbox lists events and messages that can be generated due to many possible reasons.
The Global Eventbox has a global scope and lists all real-time events. Normal Eventboxes are available mainly for retrieving historical events.
All events and messages belong to either the System, Spacecraft or User category, depending from where they originate. Their type gives more information about the severity level and is one out of the following list: Success, Informational, Warning, Error or Scheduled or can have a custom type for further differentiation. The Global Eventbox can also be used as a reminder through its capability to schedule events and to combine them with an audition profile and/or a notification profile. Being able to retrieve events even from remote workstations makes the Eventbox a useful tool to investigate problems.

How do I...

Lock a Display Window

SatView™ provides the capability to lock/unlock display windows in order to prevent an unauthorized close action.

  1. Activate the display window to be locked.
  2. Open the Lock Display Window Dialog Box by opening the Windows button's menu inside the Window panel of the System ribbon bar category and selecting the Lock/Unlock... item.
    Note:
    If the display window is locked already it must be unlocked first before any new lock can be established.
  3. Enter the password into the Password entry field. Confirm the password by entering it once again into the Confirmation field.
    Add optional information about the user, contact information and the reason for the lock into the corresponding edit fields.
  4. Push the OK button.
Note:

Only display windows with a local scope can be locked/unlocked.

Unlock a Display Window

SatView™ provides the capability to lock/unlock display windows in order to prevent an unauthorized close action.

  1. Activate the display window to be unlocked.
  2. Open the Unlock Display Window Dialog Box by opening the Windows button's menu inside the Window panel of the System ribbon bar category and selecting the Lock/Unlock... item.
    Note:
    If the display window is not locked the Lock Display Window Dialog Box will appear instead.
  3. Enter the correct password into the Password entry field.
  4. Push the OK button.
Note:

Only display windows with a local scope can be locked/unlocked.
