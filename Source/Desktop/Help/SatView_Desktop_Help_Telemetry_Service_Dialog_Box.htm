Telemetry Service Settings Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ can be made compatible with a variety of telemetry data provider products. By implementing the concept of virtual devices where for each physical device to be connected to needs a corresponding software driver, all device dependencies can be encapsulated in an elegant way.
In order to provide a maximum availability of telemetry data, more than one device can be added to the list of potential telemetry data providers. Whenever SatView™ requires telemetry data, it steps through that list until an enabled device responds. All these devices can be enabled or disabled and scheduled individually. Additional options allow the specification of backup devices.

How do I...

Use Telemetry Data Provider Devices

SatView™ implements the concept of virtual devices in order to communicate with ground segment equipment. This allows to switch easily between different equipment providing telemetry data.

Adding a Device

  1. Open the Telemetry Service Settings Dialog Box if it is not open yet:
    Open the Telemetry button's menu inside the Equipment panel of the System ribbon bar category and select the Settings... item.
  2. Choose the desired telemetry data provider product inside the Products combo box.
  3. Push the Add... button in order to add and configure the new product with the device configuration dialog box popping-up.

    Available Products:

    ENERTEC TT&C 3801-10

    ENERTEC TT&C 3801 Dialog Box
    1. Enter the name of the device into the Name edit field.
    2. Add the IP address for the device into the IP Address edit field and adjust the TCP/IP port number in the Port spin box.
      Optional:
      • Change the buffer size and timeout interval through the Buffer and Timeout spin boxes.
      • Add a filter by specifying an Application Process Identifier (APID) and/or Virtual Channel Identifier (VCID).
        Select the corresponding check boxes and adjust the numbers inside the APID and VCID spin boxes.
    3. Adjust the epoch with the documentation of the device manufacturer by adjusting the CDS Epoch spin box.
      For the ENERTEC TT&C 3801 devices this is: January 1, 1958.
    4. Adjust the time error correction by selecting the Error check box and inserting the necessary time interval.
      This amount of microseconds is used to adhere with the interval caused by the transmission delay from the spacecraft to the ground station (including delays caused by the ground segment). A positive value is subtracted from the time delivered by the ENERTEC TT&C 3801 device before time stamping the telemetry data. The opposite happens for a negative value.
      Optional:
      • Mark the Delta check box and insert a time interval to be added to the error correction.
    5. Specify the time interval after that the system clock should be correlated with the ENERTEC TT&C 3801 device. This can be achieved by selecting the Adjust the internal clock at an interval of n minutes check box and by entering the number of minutes n to be used as update interval.
    6. Push the OK button.

Removing a Device

  1. Open the Telemetry Service Settings Dialog Box if it is not open yet:
    Open the Telemetry button's menu inside the Equipment panel of the System ribbon bar category and select the Settings... item.
  2. Choose the telemetry data provider product inside the Product combo box.
  3. Select the device to be removed from the available telemetry data provider devices by selecting it inside the Devices list box and by pushing the Remove button.
  4. Push the OK button.

Specifying a Backup Device

  1. Open the Telemetry Service Settings Dialog Box if it is not open yet:
    Open the Telemetry button's menu inside the Equipment panel of the System ribbon bar category and select the Settings... item.
  2. In the Product combo box select the brand of the telemetry data provider device.
  3. Select the telemetry data provider device those backup behavior should be specified in the Devices list box.
  4. In the Backup group box choose one of the following options:

    Option

    Description

    Continuously

    Retry the active device continuously without using any other available device.

    Retry n times

    Retry the active device n times before eventually issuing a failure message (see the Audit Failures option).

    Use device x

    Use the device x if the active one is not (anymore) available.
    Select the corresponding device in the associated combo box.

    Use any other ready device

    Use any other ready device if the active one is not (anymore) available.
  5. Enable the time-out option in order to change the device after a certain time of inactivity by marking the Timeout check box and entering the number of seconds to wait into the associated edit field.
    Note:
    A period of inactivity is defined as time interval with no data traffic on a certain device.
  6. Mark the Audit Failures check box, if desired, to enable the issue of a message informing about a device unavailability.
  7. Push the OK button.

Monitoring the Device Status

  1. Right-click on the status bar of the application to make its floating menu appear.
  2. Enable the Telemetry Device Pane item in order to make the status of the current telemetry data provider device visible inside a pane.
    Displays the current status of the selected telemetry data provider device and also if SatView™ currently is connected to it. The information displayed in this pane is dependent on the kind of telemetry data provider. Some status information however is common to all such devices:

    Status

    Description

    No Device

    No telemetry data provider devices are specified.

    Not Ready

    The current telemetry data provider device is not available.

    Ready

    The current telemetry data provider device is connected to the associated equipment but no data is received.

    Offline (Disabled)

    The active telemetry data provider device is currently disabled.

    Offline (Scheduled)

    The active telemetry data provider device is scheduled to be offline at this moment.

    The associated tool-tip shows more specific information about the current telemetry data provider device.

Scheduling Devices

  1. Open the Telemetry Service Settings Dialog Box if it is not open yet:
    Open the Telemetry button's menu inside the Equipment panel of the System ribbon bar category and select the Settings... item.
  2. In the Product combo box select the brand of the telemetry data provider device to be scheduled.
  3. Select the telemetry data provider device to be scheduled in the Devices list box.
  4. In the Schedule group box choose one of the following options:

    Option

    Description

    Always

    Use the telemetry data provider device at any possible time.

    Daily between t1 and t2

    Use the device daily between t1 and t2 only.

    From t1 to t2

    Use the device between t1 and t2 only.

    At t for a period of n seconds at an interval of i seconds

    Periodically use the device every i seconds for a period of n seconds starting at time t.
  5. Select the Disabled check box, if required, in order to prevent the device from being used.
  6. Mark the Audit check box, if desired, to enable the issue of a message informing that a device scheduling has occurred.
  7. Push the OK button.

Prerequisites:
The privilege 'Changing the telemetry service settings' must be enabled to modify the configuration of the telemetry service and to add, configure or remove available telemetry data provider devices.
