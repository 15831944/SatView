Telecommand Displays

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ supports various types of telecommand related displays. Some of them allow the release of telecommands (including the ability to replay the telecommand history), others help to diagnose anomalies or reflect the on-board memory or status of the satellite.

Telecommand Sources Displays

Type

Description

Manual Stack

The TC Manual Stack is the major command source of SatView™. It is designed to manually load telecommands or telecommand sequences from the database and to eventually edit them before release. All telecommands need to be dispatched before they can be released by either a manual or automatic dispatch condition.
The TC Manual Stack supports interlocks, grouping and blocking links as well as the automatic rejection mode and release wait mode. Control over all operational aspects like Pre-transmission Validation (PTV), Pre-execution Verification (PEV) or Command Execution Verification (CEV) checking, running modes etc. is provided as well together with the capability to optionally log all activities concerning the configuration changes of the TC Manual Stack.

Automatic Stack

The TC Automatic Stack is another command source, similar to the previous one but dedicated to the automatic release of telecommands. These telecommands usually are generated by a Mission Planning System (MPS) and imported by schedule or manually. The same processing services as in the TC Manual Stack are supported but the user interface is extended with an additional pane listing the released or failed telecommands.

Procedures Monitor

The TC Procedures Monitor controls the execution of telecommand procedures. A telecommand procedure is somewhat similar to a sub-routine that can be called at a certain time or at a specific event with a predefined number of arguments. It can contain one or more threads performing all kind of computations that, in the end, may result in the submission of telecommands to the central command handler. A custom language, the Telecommand Procedure Language (TPL) has been developed to ease the development of such procedures.
The TC Procedures Monitor offers extended scheduling capabilities and also allows the triggering of a telecommand procedure through a telemetry event. In addition, it supports several execution modes (including manual mode) and has extended alert capabilities.

Telecommand History Displays

Type

Description

Command History Display

The TC History Display provides the ability to view the global commanding history archives and has advanced retrieval and filtering options: The Global TC History Display lists all released or failed telecommands in real-time mode whereas those with a local scope allow manual retrievals.

Telecommand Diagnostics & On-Board Software Displays

Type

Description

Diagnostics Oversampling Dump Display

The TC Diagnostics Oversampling Dump Display provides the capability to monitor oversampled telemetry parameters and assists in generating the telecommands necessary to control this kind of diagnostic mode.

On-Board Buffer Queues Display

The TC On-Board Buffer Queues Display reflects the status of the on-board telecommand buffer queues. It offers an overview over all on-board queues as well as a detailed representation of their contents. In addition, assistance for the generation of telecommands necessary to control those buffer queues is granted too.

On-Board Memory Images Display

The TC On-Board Memory Images Display visualizes the state and contents of the on-board processor memory areas. Dumps are detected automatically (on demand) and are updated accordingly in the database. Customizable versioning algorithms allow an effective management of those dumps and an integrated analyzing tool facilitates the comparison of (dumped) memory images. Help is also provided with the generation of the telecommands necessary to uplink memory images or to request a dump from on-board memory.

General Purpose Displays

Type

Description

Eventbox

An Eventbox lists events and messages that can be generated due to many possible reasons.
The Global Eventbox has a global scope and lists all real-time events. Normal Eventboxes are available mainly for retrieving historical events.
All events and messages belong to either the System, Spacecraft or User category, depending from where they originate. Their type gives more information about the severity level and is one out of the following list: Success, Informational, Warning, Error or Scheduled or can have a custom type for further differentiation. The Global Eventbox can also be used as a reminder through its capability to schedule events and to combine them with an audition profile and/or a notification profile. Being able to retrieve events even from remote workstations makes the Eventbox a useful tool to investigate problems.

How do I...

Lock a Display Window

SatView™ provides the capability to lock/unlock display windows in order to prevent an unauthorized close action.

  1. Activate the display window to be locked.
  2. Open the Lock Display Window Dialog Box by opening the Windows button's menu inside the Window panel of the System ribbon bar category and selecting the Lock/Unlock... item.
    Note:
    If the display window is locked already it must be unlocked first before any new lock can be established.
  3. Enter the password into the Password entry field. Confirm the password by entering it once again into the Confirmation field.
    Add optional information about the user, contact information and the reason for the lock into the corresponding edit fields.
  4. Push the OK button.
Note:

Only display windows with a local scope can be locked/unlocked.

Unlock a Display Window

SatView™ provides the capability to lock/unlock display windows in order to prevent an unauthorized close action.

  1. Activate the display window to be unlocked.
  2. Open the Unlock Display Window Dialog Box by opening the Windows button's menu inside the Window panel of the System ribbon bar category and selecting the Lock/Unlock... item.
    Note:
    If the display window is not locked the Lock Display Window Dialog Box will appear instead.
  3. Enter the correct password into the Password entry field.
  4. Push the OK button.
Note:

Only display windows with a local scope can be locked/unlocked.
